# ADR 0007: Engagement Reporting Export

## Status

Accepted

## Context

Operators need engagement exports before hardware-dependent packet capture storage is complete. The frontend also needs stable report request, status, and download contracts.

## Decision

Report requests are persisted as engagement-scoped records and generated by the backend worker boundary. V1 supports `jsonl`, `html`, `pdf`, and `pcap` formats with bounded reads from events, alerts, and audit logs.

Ready reports expose a short-lived HMAC-signed download URL and still require an authenticated session. Audit exports omit raw tool output references; PCAP exports produce an empty capture container until packet artifact storage is implemented.

## Consequences

The frontend can build the Stage 8 reporting flow against stable contracts without waiting for Raspberry Pi packet capture validation. Later storage work can replace the inline artifact storage behind the same API shape.
